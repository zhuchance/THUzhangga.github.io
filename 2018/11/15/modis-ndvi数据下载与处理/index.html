Modis数据下载

Modis查询结果获取

Modis数据可从多个网站下载，这里从NASA的LAADS DAAC下载：https://ladsweb.modaps.eosdis.nasa.gov/

界面如下：

以我们需要的1km精度、月平均的NDVI数据为例，需要分四步走：

  • 选择产品。在Sensor中应该选择MODIS Terra。

再选择一个collection。经过一番寻找发现我们需要的数据在6中，再在右侧选择具体数据即可。

  • 选择时间段
  • 选择地点
  • 文件查询

链接批量免流量下载

查询结果如下：可以将查询结果转存csv，也可以通过链接直接下载。由于文件数太多，我只能通过csv中的链接批量下载。

csv中结果如下。关键在于fileUrls。上图红框中的下载链接如下：

https://ladsweb.modaps.eosdis.nasa.gov/archive/allData/6/MOD13A3/2018/001/MOD13A3.A2018001.h28v06.006.2018033223757.hdf

可以发现下载链接的前面部分就是https://ladsweb.modaps.eosdis.nasa.gov/，而后面就是csv中的fileUrls，拼接两者就能得到有效的下载链接。

初步计算我需要下载的2800个文件大小为50G，一方面国内下载速度慢，另一方面流量耗不起。于是我用VPS下载，通过wget以100M/s的速度下载后再以1M/s的速度上传到清华网盘，再回到墙内以10M/s的速度同步。整个流程虽然受上传速度限制，但是也不算慢，关键是不费流量。具体技术路线如下：

  1. 首先在本地编程，上传到github。https://github.com/THUzhangga/modis_download/

    关键在modis_download.py，也就是读入csv数据，通过https链接头构造下载链接，然后用os操作wget进行下载。

  2. 通过ssh登录VPS，用git同步数据和程序。上面用到pandas，事实上csv也可以按一般的txt文件读入，不过我已经安装anaconda。。。在Terminal中通过 python modis_download.py就可开启下载了

  3. 配置清华网盘。我之前已经在VPS上装了gnome的desktop，所以直接装的桌面端。参照https://help.seafile.com/en/syncing_client/install_linux_client.html进行配置即可。没装Desktop也可以安装command-line client，参照官方文档配置即可。
  4. 通过VNC Viewer登录VPS，然后手动将下载的文件夹加入同步文件夹。然后就是等它上传了，大概十几个小时完成下载。

以上步骤比较曲折，如果不在乎流量，可以在windows中通过powershell下载，或者在虚拟机里中Linux系统下载。可参考https://stackoverflow.com/questions/4619088/windows-batch-file-file-download-from-a-url

但是如果有2800个文件将如何？一个一个打开不可行，只有通过程序批量处理。

万能的Python果然有轮子：https://hdfeos.org/software/pyhdf.php

以PyHDF为例：

  • 安装：实测无法使用pip或conda安装，只能在https://www.lfd.uci.edu/~gohlke/pythonlibs/下载whl文件，然后手动`python install xxx.whl`。安装过程中似乎有些依赖项，联网环境下会直接下载安装。

  • 使用：参照https://blog.csdn.net/dym755833564/article/details/7814303。在Spider中运行：

    dt.select()函数可选取一个图层。另外可以通过dt.attributes()查看hdf文件的属性。点进去可以查看各个key的value。

    比较关键的是StructMetadata.0。NDVI所在图层名是'1 km monthly NDVI'，这也是我在程序中所选取的。此外，图层的范围（XY坐标）也就是UpperLeftPointMtrs和LowerRightMtrs决定的矩形框，大小是1200×1200，时间段是其实由文件名决定了：A后面的字符串是年份+第多少天。

    知道了时间、区域、NDVI值，后续的数据处理，比如某地连续5月NDVI值归一化结果（z值）等都可自动计算。当然还是要编程…唉，又得掉头发了
