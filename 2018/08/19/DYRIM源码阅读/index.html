DYRIM源码阅读

Posted on 2018-08-19

DYRIM源码中包含以下工程：

猜测其中GlacialSnow为产雪模型，XAJ为新安江模型，WaterYield为产水模型，那么NewRouting为河网提取模型。

NewRouting

头文件与源文件对应，包括：

CoorTrans

为坐标转换程序，可以实现经纬度坐标和xy坐标相互转换，按下不表。

CpuUsageCount

为计算Cpu使用情况程序，按下不表。

DataStruct

为各种数据结构，包含

  • GravityEvent 重力侵蚀结构体。

  • BSCode 单元坐标结构体

  • Para 河段的固定参数结构体，包括波速系数、坦化系数、坡度、长、曼宁系数、边坡系数、角度、汇水面积、土质（D50，土地利用类型，土壤类型）、孔隙率、渗透率、左右源坡面的坡度长度面积、A、上下游高程

    大卫添加了ErosionK1,K2, Beta1, Beta2

  • TreeNode结构体，代表一条河段。内含

    • TaskCount：上游未计算或计算中河段数
    • UnderCalc：是否正在被计算
    • StralherOrder：级别

  • XAJParameter：新安江模型参数

  • sStatus结构体：

  • HSParameter：高边坡产流模型。

  • List类：

    • 在ListNode类上派生。该类包括bsl, bsv, regionindex, sorder, tmp
    • 函数包括insert, remove, find, N_find, first, free
      • find：找节点，返回非零值SOrder
      • N_find，根据length（bsl）和value(bsv)寻找，返回的是节点。

  • TreeList，包括：

    • TreeList类。一个TreeList代表一个RegionIndex。注意TeeList内部节点为TreeNode结构体
    • 该类内部各种代码应该是用于对河道构造树，然后计算每个单元的regionindex。包括：
      • initialize，初始化。
      • add：增加一条河段，包括BSCode坐标信息，Para参数信息，SOrder分级。似乎增加的过程还比较麻烦。如果是树的根，则需要各种赋值。如果树已经有根，先根据BSCode找到该插入的位置，也就是父节点。同时需要标记计算的一线，还要向上至更新TaskCount，得到依赖上游的河段
      • isCnnNode，判断在Regionconnection中的连接关系，补充判断其上游区域是否在本次计算任务内。
      • GetBranch：计算Min。多义：从头节点出发，找到一个介于Min和Max之间的树
      • ExpandTo：向ToList内存区展平
      • FinishBranch：当算完一个子任务时（普通的子任务，RegionIndex并没有算完），这个FinishBranch负责该普通任务算完后其所在RegionIndex的/TaskCount和UnderCalc的信息更新
      • FindParentPosit：找到父节点位置

functions

  • TreeList GetBranchInForest(TreeList* TotalList, int tmpUnitSize, int MinUnitSize, int GradeTwoCount)，作用是在一个森林里找到一个足够大的子树。如果没找到，规模减半，重头开始
  • bool IsEmptyForest(TreeList* TotalList, int GradeTwoCount)：只要存在序号小于GradeTwoCount（当前不同regionindex的个数）的空节点，即为空森林
  • template<class T>void ZeroFill(T* Qs, long DataCount)：将数组元素置0

AVPM

扩散波及重力侵蚀模型，主要包含AVPM类

  • 常数：g, gamma, nu=1e-6, fuzzpro(模糊概率积分值),大小为122*7
  • ssmethod, mstep, sccd？
  • nano
  • 关键参数：Q用来储存水流，Qout用来储存出口水流。

  • Calc：求解扩散波方程的主计算函数。计算流程如下：

    • 上游来流混合，同时检验产沙上限
    • 求空间步长数（需要用到最大流量及最大波速）
    • 空间步长循环：
      • 非第0层Qout赋给Q
      • 第一个时间点不计算
      • Q很小时改变糙率（完成阻力分解）
      • 根据流量计算水深：$Q=AC\sqrt{RJ}$，其中$A=mh^2$, $C=\frac{1}{n}R^{1/6}$, $R=\frac{mh}{2\sqrt{1+m^2}}$，代入即可解得：
      • 先用Q[0]计算h1，用Qout[0]计算h3。
      • 开启时间循环：
        • Qout[t]=Q[t]
        • 计算h0,h2,以及新的h1,h3. 注意h0 对应(i,n-1)，h1对应(i,n)，h2对应(i+1, n-1),h3对应(i+1,n)
        • 计算u0~u3, c0~c3,从而求得$\bar{c}$，$\bar{B}$，进而再由$\frac{\partial Q}{\partial x}$求得$\frac{\partial h}{\partial x}$以及能坡$J$
        • 反过来由能坡$J$又能求新的u0~3,c0~3,B0~3,从而有新的$\bar{c}$，$\bar{B}$。看收敛就是看前一个$\bar{c}$与新的$\bar{c}$之间的差距。
        • 如果波速不收敛，则采用牛顿迭代。
        • 完成后采用空间四点偏心离散求解，得到新的Qout[t]。
        • 迭代至Qout[t]收敛
          • 用Qout[t]计算h3，用之前的h0,h1,h2得$\bar{h}$。$\frac{\partial Q}{\partial x}$不重算，得到$\frac{\partial h}{\partial x}$以及能坡$J$。。。
    • 上游水沙演进到出口，再叠加本单元水沙到出口

  • Getss：

    • 计算含沙量。输入包括体积含沙量，水深，流速和沉速指针（？）
    • 这里考虑了Svm，极限体积比含沙量，只与D50相关。

  • GetssFei：

    • 计算含沙量。采用费祥俊公式。

  • SediCalc:

    • SSin和SSout，即进口和出口的挟沙力。
    • 如果出口挟沙力大于含沙量且在涨水，Sout[t]采用某公式计算；否则采用另外的公式计算。
    • 如果考虑重力侵蚀且崩塌量大于0且为淤积（由上面步骤确定），则要考虑崩塌量的变化。

  • initialize：

    • 初始化变量

  • finalize:

    • 回收内存空间
  • 重力侵蚀的函数：
    • InitGravityPara：初始化重力侵蚀参数
    • InitGullyShap：同时给左坡右坡初始化，为了后边调整水位
    • ConvertTao
    • CalFs
    • CalFuzzProbi
    • HowMuchFailed
  • SaveStatus：
    • 保存到status表中的数据
    • 类似于移动光标。但好像只保存了channelero和gravityero?
